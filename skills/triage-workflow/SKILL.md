# 目标

把分散问题整理成“可排序、可指派、可追踪”的处理清单。

# 适用场景

- 用户要求“分诊、排优先级、梳理待修问题”。
- 需要快速识别高优先级问题并安排处理顺序。

# 执行流程

1. 收集问题
   - 统一问题描述格式：现象、触发条件、影响对象。
   - 去重并合并同类问题。

2. 评估维度
   - 严重度：阻断/高/中/低
   - 影响范围：单功能/多模块/全局
   - 可复现性：稳定复现/偶发
   - 修复成本：小/中/大

3. 确定优先级
   - 默认规则：高严重度 + 大范围 + 稳定复现 = 高优先级。
   - 标记“立即处理”“本迭代处理”“可延期”。

4. 形成执行清单
   - 每项包含：负责人建议、预估工作量、验证口径。
   - 输出建议处理顺序与依赖关系。

# 工具映射建议

- 汇总背景信息：`read_file`、`search_workspace`
- 生成分诊清单：`write_file`
- 若需验证现象：`run_terminal`

# 输出结构

1. 分诊标准（本次使用）
2. 问题分级结果
3. 建议优先级队列
4. 每项下一步动作
5. 风险与阻塞项
