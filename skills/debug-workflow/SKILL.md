# 目标

将问题修复为“可复现、可解释、可验证”的结果，避免只修表象。

# 适用场景

- 用户描述“报错、异常、崩溃、不生效”。
- 需要从症状出发定位根因并给出最小改动。

# 执行流程

1. 复现与收敛
   - 先复述现象、触发条件、预期行为。
   - 用最小输入或最短路径复现问题。
   - 信息不足时，先列出假设并标注不确定项。

2. 定位根因
   - 先读入口与调用链关键文件，再扩大范围。
   - 区分“根因”和“连带症状”，优先修根因。

3. 最小修复
   - 只改与根因直接相关代码，避免无关重构。
   - 保持兼容性；若需改行为，先说明影响面。

4. 修后验证
   - 执行最贴近问题的验证命令（测试/构建/类型检查）。
   - 说明结果、覆盖范围和剩余风险。

# 工具映射建议

- 看目录与范围：`list_dir`
- 读关键文件：`read_file`
- 关键词定位：`search_workspace`
- 修改代码：`write_file`
- 运行验证：`run_terminal`

# 输出结构

1. 问题复现结论
2. 根因说明
3. 修改点与意图
4. 验证步骤与结果
5. 剩余风险与后续建议
