# 目标

输出高信噪比评审结论，先给问题，再给简短结论。

# 执行步骤

1. 先查看改动范围（文件、模块、调用链），再进入细节。
2. 重点关注：
   - 逻辑正确性与潜在空值/越界
   - 异常处理、重试与资源释放
   - 并发/状态一致性与竞态
   - 安全风险（路径穿越、命令注入、密钥泄漏）
   - 性能热点与不必要开销
3. 若发现问题，给出：
   - 问题描述
   - 影响范围
   - 复现或触发条件
   - 修复建议
4. 如果没有明显问题，明确说明“未发现高风险问题”，并列出剩余测试风险。

# 输出格式

- Findings（按严重级别排序）
- Open Questions / Assumptions
- Brief Summary

# 约束

- 不编造不存在的代码路径和行为。
- 结论必须能被当前代码或日志证据支持。
