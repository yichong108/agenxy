# Frontend Slides PPT Controlled

在 `frontend-slides` 工作流基础上执行，但必须额外满足本技能的约束项。  
若与其他默认行为冲突，以本技能为准。

## 触发条件

当用户出现以下任一意图时启用本技能：

- 生成或改造 `ppt`/`pptx`/演示文稿
- 将 PPT 转成 HTML slides
- 需要固定横向展示与稳定翻页操作
- 需要“点击后逐步显示内容”
- 需要“主题切换”与“指示器显示控制”

## 执行流程

1. **优先在当前工作目录（项目根 / 会话工作区）下的 `frontend-slides/` 文件夹查找**：若存在且包含所需配套文件（如 `SKILL.md`、`html-template.md`、`viewport-base.css`、`scripts/` 等），则一律以该目录为技能与资源来源，无需再视为“未安装”。
2. 仅当当前目录下没有 `frontend-slides/`，或目录存在但缺少关键文件时，再检查全局/其他路径是否已安装 `frontend-slides`。
3. 再按其核心流程完成需求采集与内容生成。
4. 生成前，强制注入本技能的“配置项”和“交互逻辑”。
5. 输出后，确认默认值与开关行为符合本技能约束。

## 左下角工具栏规则（强制）

- 当鼠标进入页面左下角热点区域时，必须显示一条横向工具栏。
- 工具栏显隐必须采用透明度过渡（opacity fade）实现，不使用位移突变或直接 `display: none` 硬切。
- 工具栏默认隐藏态为低透明或全透明；鼠标移入工具栏后必须保持稳定显示，不得抖动或自动收起。
- 仅当鼠标同时离开“左下角热点区域”和“工具栏区域”时，工具栏才允许淡出隐藏。
- 隐藏态必须不可误触（如配合 `pointer-events: none`）；显示态恢复可交互。
- 可配置项必须集中放在该工具栏中，不再要求通过控制台/命令行单独收集。
- 工具栏至少提供当前允许配置的全部选项：主题切换、分步显隐开关、指示器显隐开关、右上角说明显隐开关、转场动画切换。
- 固定约束项（如固定横向）不作为工具栏选项展示。
- 工具栏交互结果写回生成配置（如 `deckConfig`），并实时作用于页面行为。
- 工具栏全部视觉样式必须跟随当前主题变量：背景、文字、边框、阴影、分隔线、按钮默认态/悬浮态/激活态/禁用态。
- 禁止工具栏使用与主题脱节的固定色值皮肤；主题切换后工具栏样式需同步即时更新。

## 必须实现的 5 项约束

### 1) 固定横向与翻页操作

- 页面方向固定为 `horizontal`，不提供方向切换选项。
- 必须支持键盘方向键翻页（至少 `ArrowRight` 前进、`ArrowLeft` 后退）。
- 必须支持鼠标点击进入下一步/下一页（如点击页面空白区或指定可点击区域）。
- 键盘与鼠标触发“下一步”时，若已开启分步显隐则先遵循 reveal 优先级，再进入下一页。

### 2) 一页内容铺满

- 每页必须完整占据视口（`100vh`/`100dvh`），不允许页内滚动
- 内容密度超限时，必须自动拆分为多页，不得缩小到不可读
- 禁止出现“需要滚动才看全”的页面

### 3) 分步显隐动画（点击逐步显示）

- 每页的可分段内容（要点、卡片、图文块）必须支持“按一下显示一部分”
- 分步显隐默认关闭（`revealOnStep: false`），可在左下角工具栏开启
- 当分步显隐开启时，默认进入页面仅显示第一层内容或标题层
- 每次用户触发下一步（键盘、点击或翻页手势）先推进当前页 reveal 阶段；当前页 reveal 完成后再翻到下一页
- 动画应简洁可读（如 fade/slide），并支持 `prefers-reduced-motion`

### 4) 主题切换（固定提供 20 种）

- 必须提供以下 20 个主题供选择（通过左下角工具栏展示与切换）：
  1. `minimal-clean`（极简白）
  2. `cyber-neon`（暗夜霓虹）
  3. `academic-blue`（学术蓝）
  4. `executive-gold`（商务灰金）
  5. `forest-nature`（自然森系）
  6. `zen-japanese`（日式留白）
  7. `chinese-classic`（国风雅韵）
  8. `deep-space-tech`（科技深空）
  9. `vintage-editorial`（复古报刊）
  10. `pastel-soft`（马卡龙柔和）
  11. `mono-contrast`（高对比黑白）
  12. `aurora-gradient`（渐变流光）
  13. `glassmorphism`（玻璃拟态）
  14. `neumorphism`（新拟物）
  15. `illustrative`（插画手绘）
  16. `campus-learning`（教育课堂）
  17. `documentary`（纪录片风）
  18. `gallery-style`（艺术展览）
  19. `data-dashboard`（数据驾驶舱）
  20. `festival`（节日庆典）
- 默认主题必须是 `deep-space-tech`
- 使用 CSS 变量组织主题，切换时不改动内容结构

### 5) 指示器默认显示（提供切换）

- 指示器默认必须为显示状态。
- 必须提供指示器显示/隐藏切换选项：`showIndicators: true | false`。
- 当 `showIndicators = false` 时，不得渲染页码点、导航点或任何指示器占位元素。

### 6) 右上角说明显隐（工具栏控制）

- 右上角说明区域必须提供显示/隐藏能力。
- 该开关必须放在左下角工具栏中，不得只依赖键盘快捷键或代码硬编码。
- 默认状态必须为隐藏，但必须在工具栏中可见且可切换。

### 7) 转场动画切换（工具栏控制）

- 必须提供转场动画选项，并在左下角工具栏中可切换。
- 至少提供 3 种转场模式（例如：`slide`、`fade`、`zoom`）。
- 必须提供 `none` 或 `instant` 模式，便于弱动画或演讲场景。
- 转场切换后应立即影响后续翻页，不得要求刷新页面。

## 布局多样性约束（强制）

- 同一份演示文稿中，每一页必须采用不同的版式结构，不得整套页面仅替换文案与图片。
- 至少在以下维度中变化两个以上：信息分栏、主视觉位置、标题区形态、卡片组织方式、图文比例、留白节奏。
- 相邻两页不得使用同构布局（例如同样的标题 + 三列卡片 + 底部注释结构）。
- 若内容导致可选布局不足，必须通过拆页或重组信息来保持版式差异。

## 视觉素材约束（强制）

- 每一页都应适当加入图片、插图或图标元素，不得整页纯文字堆叠。
- 在不影响信息表达的前提下，适当提供插画配图以增强理解与记忆点。
- 图片/图标必须与该页主题信息相关，优先承载解释作用而非纯装饰。
- 素材密度需适中，不得压缩正文可读性或干扰讲解节奏。
- 对图标风格保持全稿一致性（如线性/面性风格统一），避免混搭失衡。
- 可按页面内容适当添加背景图，以增强语义氛围与视觉层次。
- **整页背景图必须铺满整页视口（强制）：** 作为单页全幅背景时，图像层必须覆盖该页 `100% × 100%` 可视区域，不得出现未填充的留白条（letterboxing）。CSS 侧默认采用 `background-size: cover`、`background-position: center`（或等效逻辑）；若使用 `<img>` 作为底层全幅图，须 `width/height: 100%`、`object-fit: cover`，并与幻灯片同尺寸定位（如绝对定位铺满 `.slide`）。禁止用 `contain`、固定小尺寸或仅局部贴图冒充整页背景。
- 背景图不得喧宾夺主；必须通过遮罩、模糊或透明度控制确保前景文字清晰可读。
- 相邻页面背景图风格应协调但不雷同，避免重复素材造成审美疲劳。
- 推荐硬参数（默认执行）：
  - 背景遮罩透明度不低于 `0.35`（深色背景可提高到 `0.45-0.60`）。
  - 背景图模糊建议 `2px-6px`（信息密集页优先使用更高值）。
  - 正文区域优先放置在背景低频区域，避免穿过高对比细节。

## UI 标准参考（强制）

- 视觉与交互设计必须参考成熟设计体系，不得仅凭主观“美观”判断。
- 默认参考优先级：`Material Design 3`、`Apple Human Interface Guidelines`、`Microsoft Fluent`（按场景择优吸收）。
- 至少落实以下可检查标准：
  - 视觉层级清晰：标题、正文、注释、弱信息有稳定层级，不可混乱同权重。
  - 对齐与间距一致：使用统一间距节奏（如 4/8pt 系统），避免随机留白。
  - 对比度达标：文本与背景应满足可读性要求（优先对齐 WCAG 对比度建议）。
  - 交互可预期：悬停、点击、切换反馈一致，不出现“可点击但无反馈”元素。
  - 状态明确：工具栏显示/隐藏、开关启停、当前页状态都有清晰视觉反馈。
- 若用户提供品牌规范（色板/字体/语气），品牌规范优先于通用设计体系。

## 每页讲稿备注约束（强制）

- 每一页必须提供讲稿备注（speaker notes），不得缺省。
- 每页讲稿备注应控制在约 30 秒可讲完（建议约 90-140 字中文，按正常语速）。
- 讲稿备注内容需与该页视觉内容一一对应，避免泛化复述。
- 讲稿备注应包含“开场句 + 关键信息 + 过渡句”，保证演讲连贯。
- 讲稿数据必须汇总写入 `window.__META__.notes`。
- `window.__META__.notes` 必须按页顺序存储（索引与页码一一对应），长度必须等于总页数。
- 若输出格式支持结构化字段，仍可保留每页独立 `notes` 字段，但不得替代 `window.__META__.notes`。

## 推荐配置接口

生成代码时优先使用统一配置对象，至少包含：

```js
const deckConfig = {
  orientation: 'horizontal', // 固定横向，不提供切换
  theme: 'deep-space-tech', // 20 个固定主题之一
  showIndicators: true, // 默认显示，允许切换
  showTopRightNotes: false, // 右上角说明默认隐藏（工具栏可切换）
  transitionStyle: 'slide', // slide | fade | zoom | none
  revealOnStep: false // 分步显隐默认关闭（工具栏可开启）
}
```

## 页面元信息约束（强制）

- 生成的 HTML 页面必须定义全局对象变量：`window.__META__`。
- `window.__META__` 必须至少包含属性：`totalPageNum`。
- `window.__META__` 必须包含属性：`notes`（每页讲稿数组）。
- `totalPageNum` 必须等于当前文稿真实总页数（即 `.slide` 数量）。
- `notes.length` 必须等于 `totalPageNum`，且每一项对应一页讲稿。
- 若页面已存在 `window.__META__` 其他字段，写入 `totalPageNum` 时不得破坏已有字段。
- 推荐写法示例：

```js
window.__META__ = {
  ...(window.__META__ || {}),
  totalPageNum: document.querySelectorAll('.slide').length,
  notes: ['第 1 页讲稿...', '第 2 页讲稿...']
}
```

## 交互优先级规则

当 `revealOnStep = true` 时，用户触发“下一步”应遵循：

1. 若当前页仍有未显示内容，优先 reveal 下一段
2. 若当前页已 reveal 完整，再切换到下一页

当“上一步”时同理：

1. 若当前页可回退 reveal，则先回退 reveal
2. 否则再切换上一页

## 验收清单（生成后必须自检）

- [ ] 方向固定为 `horizontal`，且不存在方向切换选项
- [ ] 支持键盘方向键与鼠标点击进入下一步/下一页
- [ ] 左下角工具栏使用透明度方式显示/隐藏，且交互稳定无闪烁
- [ ] 工具栏所有样式与状态均跟随当前主题，并在主题切换后即时同步
- [ ] 每页占满视口，无页内滚动
- [ ] 分步显隐默认关闭，开启后生效且优先级正确
- [ ] 存在 20 个固定主题选项并可在工具栏切换，默认 `deep-space-tech`
- [ ] 指示器默认显示，且存在工具栏显隐切换选项
- [ ] 右上角说明存在工具栏显隐开关，且切换行为生效
- [ ] 工具栏提供转场动画选项（含弱动画模式），切换后后续翻页立即生效
- [ ] 每页均提供约 30 秒讲稿备注，且与页面内容对应
- [ ] 讲稿已写入 `window.__META__.notes`，顺序与页码一致，且长度等于总页数
- [ ] 每页均有适量且相关的图片/图标素材，且不影响可读性
- [ ] 背景图使用适度，且前景文字对比度与可读性达标
- [ ] 凡整页背景图均已铺满该页视口（cover / object-fit: cover），无留白条或未覆盖区域
- [ ] 背景图满足最低可读性参数（遮罩 >= 0.35，必要时增加模糊）
- [ ] UI 实现符合参考设计体系的关键标准（层级、间距、对比度、反馈、一致性）
- [ ] 页面已定义 `window.__META__`，且 `totalPageNum` 与实际页数一致

## 失败处理

若任一约束无法满足：

1. 明确指出失败项
2. 给出最小改动修复方案
3. 修复后重新执行验收清单
