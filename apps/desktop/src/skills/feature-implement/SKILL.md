# 目标

以最小但完整的方式交付可运行、可验证的新功能。

# 执行步骤

1. 需求澄清
   - 提炼目标、输入输出、边界条件。
   - 若需求含糊，先给默认假设并在结果中标注。

2. 方案设计
   - 找到最相关的模块与调用链。
   - 优先选择对现有结构侵入最小的实现路径。

3. 实施落地
   - 先读取必要文件再修改。
   - 只改与当前需求直接相关内容，避免顺手大重构。
   - 保持风格一致与命名一致。

4. 自检与验证
   - 运行必要验证命令（类型检查 / 单测 / 构建）。
   - 说明验证结果与潜在剩余风险。

# 工具建议

- 理解阶段：`list_dir`、`search_workspace`、`read_file`
- 改动阶段：`write_file`
- 验证阶段：`shell`

# 输出格式

1. 需求理解与假设
2. 代码修改点
3. 验证步骤与结果
4. 后续可选优化

# 约束

- 不访问工作区外路径。
- 不执行破坏性命令。
- 不泄露敏感信息。
